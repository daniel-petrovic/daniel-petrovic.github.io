This site starts with a very simple goal: create a clean personal homepage now, and leave enough structure to publish technical notes later.

That makes **Jekyll** a good fit. It keeps the generated site static, the project structure understandable, and the writing workflow straightforward.

## What I want from the setup

- A homepage that feels like a personal introduction instead of a product page
- A blog that can hold short technical notes as well as longer posts
- A deployment path that works well with GitHub Pages
- Enough structure for reuse without turning the site into a big framework exercise

## Topics I expect to write about

The rough direction is software engineering with room for:

- C++ and systems-oriented programming
- Tooling and workflow improvements
- Notes from debugging, refactoring, and learning

For now, the important part is that the site is ready to grow.
